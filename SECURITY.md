# Security Policy

## Reporting a Vulnerability

**Do not open a public GitHub issue for security vulnerabilities.**

Please report security vulnerabilities through the repository's **Security** tab > **Report a vulnerability**.

This keeps the report confidential until a fix is available.

## Coordinated Disclosure Timeline

We follow a 90-day coordinated disclosure timeline:

1. You report the vulnerability privately via the Security tab.
2. We acknowledge receipt within 7 days.
3. We work on a fix and keep you informed of progress.
4. We publish a fix and a security advisory within 90 days of the initial report.
5. After the fix is released, you are free to disclose publicly.

If extenuating circumstances require more time, we will communicate that before the 90-day window closes.

## Scope

The following constitute security issues:

- Authentication or authorization bypasses
- Remote code execution
- Sensitive data exposure (credentials, tokens, PII)
- Dependency vulnerabilities with a clear exploitation path in this project

The following are **not** security issues and should be filed as regular bugs:

- Missing features
- Incorrect output that does not expose sensitive data
- Performance issues
- Cosmetic or documentation errors

## Contact

We do not accept security reports via email or PGP. Use the repository's **Security** tab > **Report a vulnerability** exclusively. This ensures all communication is tracked, auditable, and does not rely on personal contact information.
